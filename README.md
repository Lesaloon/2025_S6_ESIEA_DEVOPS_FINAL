1. Comment définiriez-vous le Devops ?
Le DevOps est une démarche qui vise à améliorer la collaboration entre les équipes de développement et d'administration système. Il se repose sur l'automatisation des processus de déploiement et de validation du code. Il permet aussi de réduire le temps de mise en production des applications.

2. Qu’impose le Devops ?
Le DevOps impose une grande automatisation des process et un language commun entre les équipes de développement et d'administration système.

3. Quels sont les inconvénients ou les faiblesses du Devops ?
Le DevOps peut être difficile à mettre en place dans des projets de grande envergure. Et une mauvaise mise en place peut entraîner des problèmes de sécurité et de performance. ( exemple au travail j'ai des pipeline qui prenent 15min )

4. Quel est votre avis sur le Devops ?
J'aime bien le DevOps car il permet de réduire le temps de mise en production des applications et assure que le code est d'une qualité acceptable.

5. Quels sont les tests primordiaux pour toute application ?
Les tests unitaires ( backend ) et les tests fonctionnels ( frontend ) sont nécéssaires pour mettre en place une bonne CI/CD.
